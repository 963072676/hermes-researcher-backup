# Hermes Session State Integrity Deck

> Draft generated by researcher tick34. Review before installation.

## Trigger

Use when any of these occurs:
- compaction summary contains a user ask with no matching transcript message;
- consecutive compactions degrade or loop;
- lock acquisition errors allow mutation to proceed;
- a session with live background work is pruned;
- closing a TUI/dashboard observer causes a messaging session to lose history;
- one routing key mints a new session on every inbound message.

## Invariants

1. **Provenance** — every summarized user ask maps to source message IDs.
2. **Single writer** — mutation requires explicit lock success.
3. **Liveness preservation** — an indeterminate active-process check keeps the session.
4. **Surface ownership** — observers cannot finalize another surface’s routing target.
5. **Bounded compaction** — degraded summaries increment the ineffective counter.
6. **Recovery fidelity** — reopen restores original history and stable session identity.

## Workflow

### 1. Freeze destructive maintenance

Pause only the session cleanup/compaction operation under investigation. Do not delete session rows or modify production state during diagnosis.

### 2. Capture a read-only snapshot

Record:
- session ID and routing key;
- `ended_at`, `end_reason`, active flag;
- message count and latest message ID;
- background process IDs/status;
- current compression lock owner/expiry;
- gateway/TUI/Desktop owners observing the session.

Never include authentication material in the snapshot.

### 3. Provenance check

For every summary line classified as a user request or pending ask:
1. extract the claim;
2. search the canonical transcript;
3. attach one or more source message IDs;
4. reject the claim if no source exists.

Pass condition: 100% of user-intent claims have a source.

### 4. Lock failure injection

Test four outcomes from the lock provider:
- true → one mutation proceeds;
- false → mutation skips;
- expected compatibility error → explicitly documented fallback only;
- unknown exception → mutation skips and emits an error metric.

Run two concurrent compactors. Pass condition: one writer, one skipped caller, one resulting history lineage.

### 5. Prune failure injection

Make the active-process callback:
- return true;
- return false;
- raise an exception.

Pass condition: only explicit false allows age-based pruning. True or exception keeps the session.

### 6. Cross-surface ownership test

Open a live messaging session in Dashboard/TUI, then disconnect the observer. Pass conditions:
- messaging session keeps the same ID;
- `ended_at` remains unset;
- next inbound message loads non-zero history;
- no empty replacement session is minted.

### 7. Compaction circuit breaker

Simulate empty/degraded summaries repeatedly. The fallback marker must count as ineffective. Pass condition: the configured threshold stops additional compression, records a terminal reason, and preserves the last good context.

### 8. Recovery readback

After each state mutation, read the canonical store again and compare:
- identity;
- owner;
- ended state;
- message count;
- active processes;
- summary source map.

## Pitfalls

- A row existing in the DB does not prove the model can reload its history.
- A process being alive does not prove the gateway adapter is ready.
- Catching an exception and continuing is unsafe for state transitions.
- TUI/dashboard ownership must not be inferred from mere observation.
